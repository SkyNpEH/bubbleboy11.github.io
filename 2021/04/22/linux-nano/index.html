    文本编辑器Text Editor，

    • 一种程序，编辑和查看纯文本文件，没有任何文本修饰

    • 不能对文字做格式处理：没有任何粗体，下划线、斜体，图形，符号或特殊字符及特殊打印格式的文本

    • 只保存文本，不保存其格式设置

    • Windows 下的记事本 纯文本文件就是 .txt 结尾的文本文档；
      macOS 下的文本编辑
      Linux 终端的文本编辑器中，有：Nano，Vim，Emacs, Gedit，Sublime，Atom; DOS 下的 edit
      在 Linux 和 macOS 下，后缀名没那么重要，不少纯文本文件是没有后缀名的

    文本处理器（文档编辑器）Text Processor

    能以编辑和查看文档，对文字做格式处理，例如：加粗，斜体，改变颜色，改变字体大小，添加超链接，加下划线，设为标题，插入图片

    这些软件只能在图形界面下使用
    Windows 下的 Word；macOS 下的 Pages
    ◆ Linux 下的 OpenOffice Writer，等等

    文本编辑器处理的文件格式和文本处理器的文件格式一般也不同

    Nano 文本编辑器，不是文本处理器 GNU Nano

    Nano is ANOther editor “nano 是另一种编辑器”
    nano 是“毫微”的意思，也就是 10 的 -9 次方

    Nano 中的键盘快捷键

    终端下部，我们可以看到有两行帮助文档
    这些快捷键组合中虽然字母是大写，但就是键盘上对应的字母键，并不需要用 Shift 键来将其变为大写

    ^ 这个符号表示键盘的 Ctrl 键
    Nano 的快捷键组合
    ^X 标明的是 Exit，按Ctrl+X 退出Nano
    Ctrl+Y：上一个屏幕
    Ctrl+V：下一个屏幕
    PaUp 和 PaDn 也分别与 Ctrl+Y 和 Ctrl+V 功能一样
    复制一整行：Alt+6
    Ctrl+K 剪切当前一整行
    Ctrl+U 粘贴刚剪切的内容

    如果需要复制／剪贴多行或者一行中的一部分，先将光标移动到需要复制／剪贴的文本的开头，按Ctrl+6（或者Alt+A）做标记，然后移动光标到 待复制／剪贴的文本末尾。这时选定的文本会反白，用Alt+6来复制，Ctrl+K来剪贴。若在选择文本过程中要取消，只需要再按一次Ctrl+6。

    Ctrl+W，然后输入你要搜索的关键字，回车确定。这将会定位到第一个匹配的文本，接着可以用Alt+W来定位到下一个匹配的文本。
    Ctrl+G：显示帮助文档
    Ctrl+O 保存所做的修改
    Ctrl+R：打开其他文件
    Ctrl+\：替换
    Ctrl+F：向前移动一格光标
    Ctrl+B：向后移动一格光标
    Ctrl+P：向上移动一行
    Ctrl+N：向下移动一行
    用键盘上的方向键（上，下，左，右）也可以控制光标移动

    先按 Esc 键，再按 X 键，帮助文档就没了

    nano file_path直接打开，如果你对 file.txt 有写的权限，那你就可以用 nano 来修改这个文件了，没有该文件的话，就会创建一个空文件

    -m：激活鼠标，鼠标可以通过点击来控制光标的位置。如果没有 -m 参数，那么鼠标在 nano 里是不起作用的
    -i：激活自动缩进
    -A：激活智能 Home 键的功能。通常状况下按下键盘的 Home 键，光标会立即跳到一行的最开始。用了 -A 这个参数
    ◆ 按下 Home 键它会智能地判断，根据一行的开始处有无缩进来跳转，如果有缩进会跳转到缩进之后而不是最开始
    nano -miA file.txt

    配置nano编辑器

    单个用户配置：

    每个 Linux 的用户都可以在自己的家目录/home/username路径下创建 .nanorc 这个文件，在每次 nano 启动前，它会读取此配置文件

    最前面有一个点.，表明这是一个隐藏文件
    ◆ Linux 中的配置文件大多以点开头，
    而且多以 rc 结尾，“rc”是“runcomm”的缩写，即“run command”(运行命令)，”具有从档案中取出一系列命令来执行的功能
    比如 Vim 的配置文件 .vimrc；
    Linux 或 Unix 的许多程序在启动时，都需要“rc”后缀的初始文件或配置文件，
    “ rc”是很多脚本类文件的后缀，常在程序的启动阶段被调用，通常是 Linux 系统启动时

    而 .bashrc 是当 Linux 的 Bash shell 启动后所运行的脚本配置文件

    配置语言以set和unset开头：
    `Code

    相当于命令后面加 -m 参数 鼠标生效

    set mouse

    相当于 -i 参数 自动缩进

    set autoindent

    激活职能home键 -A参数

    set smarthome
    `
    要保存文件，只要按下 Ctrl+O，它会提示你文件名是 .nanorc
    已经指定了文件名，直接按下回车，这三行就写入到了.nanorc中
    可以看到终端提示：[Wrote 3 lines]，表示“写入了 3 行
    按下 Ctrl+X 来退出 nano
    下次你再启动 nano 的时候，你会发现：鼠标、自动缩进、智能 Home 键也激活了

    全局配置：
    每个用户的家目录中的 .nanorc 这个文件非常实用，可以帮助你设置自己的 nano 选项
    nano 有一个全局的配置文件，是为系统上所有用户所公共调用的，也叫 nanor
    /etc/nanorc文件，这回 nanorc 前面没有点.了
    （只能被 root 用户修改）

    /etc 这个目录，它里面存放系统的配置文件。一般为所有用户共用
    如果我们要修改这个文件，建议用命令sudo nano /etc/nanorc
    在这个配置文件里，有所有可以放置在你自己的 .nanorc 中的语句
    在/etc/nanorc中把那些配置语句前面的#去掉，会对全局用户生效

    配置终端

    单个用户的终端配置文件/home/username/.bashrc

    一般来说是默认存在的，不像 .nanorc 可能还要自己创建
    nano ~/.bashrc打开家目录下的终端配置文件看看

    bash其实是一种shell， CentOS 和大部分常见的 Linux 发行版默认的 shell 程序就是Bash
    shell是外壳程序（shell 是英语“外壳”的意思），用于解释我们输入终端的各种命令的，是用户和操作系统之间的一个命令解释器，是用户与 Linux 操作系统之间沟通的桥梁

    bashrc应用于不用用户登录的图形化终端，也就是我们一直在使用的终端形式，图形化的终端的情况
    这种终端是读取 .bashrc 为配置文件的
    .bashrc 就是 Bash 这个 shell 程序的配置文件
    .bashrc 本身的语法也是 Bash 的语法，是一种脚本语言
    配置.bashrc文件可以修改终端的各种配置，如命令提示符样式，创建命令别名

    `Code

    创建别名 alias xx=’command’

    alias ll=’ls -alF’
    alias la=’ls -A’
    alias l=’ls -CF’
    `

    终端的 bash的全局配置的路径：/etc/bashrc，
    家目录下的 .bashrc 和优先级高于系统的/etc/bashrc文件高
    nano ~/etc/bashrc
    把 bashrc 文件向下拉，看到有 PS1 开头的行，设置命令行提示符的样式的，
    PROMPT_COMMAND='printf "\033]0;%s@%s:%s\007" "${USER}" "${HOSTNAME%%.*}" "${PWD/#$HOME/~}"'
    ${USER} 表示用户名（例如我的用户名是 oscar
    ${HOSTNAME} 是电脑的名称（我的情况是 oscar-laptop）
    @ 分割用户名和电脑名的那个 @

    profile配置文件

    单个用户：~/.profile，全局 /etc/profile
    nano ~/etc/profile
    vi /etc/profile
    export CLASSPATH=./JAVA_HOME/lib;$JAVA_HOME/jre/lib
    profile应用TTY1-TTY6这种需要用户登录的，非图形界面的终端的配置文件。
    profile文件会调用.bashrc，修改了 .bashrc，也就是间接修改了 profile 文件，
    profile 文件会用 profile 本身的配置再加上 .bashrc 的配置
