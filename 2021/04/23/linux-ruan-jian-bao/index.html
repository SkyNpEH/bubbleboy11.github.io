    linux 软件包

    发表于 2021-04-22 更新于 2021-06-22 分类于 操作系统

    在 Windows 下，要安装软件，一般使用“安装程序”
    一般是 .exe 结尾的可执行程序，双击即可运行，一般会把64位软件安装到 Program Files 这个文件夹下

    CentOS 下也有类似的程序，但不称之为“安装程序，而称之为“软件包”，英语是 Package
    一个软件包其实是软件的所有文件的压缩包
    二进制形式的，包含了安装软件的所有指令
    ◆ 在 Red Hat 一族里，软件包的后缀是 .rpm
    rpm 是 Red Hat Package Manager 红帽软件包管理器
    RedHat系列：Redhat、Centos、Fedora用 .rpm 的软件包
    Debian 一族（Ubuntu，Debian，等）的软件包是 .deb 为后缀

    Linux的软件包相较于windows的exe安装包文件，包含了依赖关系的管理，对于依赖的软件会自动下载。
    软件包不需要我们通过搜索引擎来找到并下载
    Linux几乎所有.rpm软件包存放在相同地方，称为软件仓库（repository）
    软件仓库绝不只是存放在一台服务器上，在全世界各地分布有软件仓库。（也就是有很多服务器）
    大部分软件仓库里的软件都是一样的，只不过是拷贝而已
    有一些特别的软件仓库，会存放其他软件仓库没有的软件
    是用户选择软件仓库，因为基本上各个软件仓库中的软件都是一样的
    一般建议用户选择离自己所在地较近的软件仓库的服务器

    dependency依赖关系：一个软件依赖其他程序，还有下层依赖关系，环环相扣

    • 很少有一个软件可以单独在 Linux 上运行，它不依赖于其他程序
    • 一个软件经常需要使用其他程序或者其他程序的片段（称之为 库
      Linux 下类似 Photoshop 的软件 GIMP，GIMP 的正常运作需要调用图片读取的库（例如读取一个JPG图片）

    管理软件仓库

    如果我们要切换软件仓库，可以通过编辑软件仓库列表文件，是系统文件，只能被 root 用户修改
    CentOS是/etc/yum.repos.d/CentOS-Base.repo

    配置yum源
    1、安装wget下载工具

    yum install wget -y

    2、进入yum源配置文件

    //进入yum配置文件夹

    cd /etc/yum.repos.d

    //备份之前的配置文件
    备份系统旧配置文件
    mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup
    mv CentOS-Base.repo CentOS-Base.repo.bak

    //阿里源
    wget http://mirrors.aliyun.com/repo/Centos-7.repo

    wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.cloud.tencent.com/repo/centos7_base.repo

    4、设置需要的镜像源为默认源
    //修改指定的yum源为默认yum源
    mv /etc/yum.repos.d/Centos-7.repo /etc/yum.repos.d/CentOS-Base.repo

    5、清空之前缓存生成新的缓存
    //清空缓存
    yum clean all
    //生成新的缓存
    yum makecache

    debian是/etc/apt/sources.list
    一般来说，这个文件中有作用的行是由以下两个指令开头：
    deb: 用于下载软件的二进制版本，大多数情况下我们用的是这个
    deb-src: 用于下载软件的源代码
    `Bash

    举例其中一行

    deb http://cn.archive.ubuntu.com/ubuntu/ bionic universe
    `
    第二个是仓库地址，第三个bionic是Ubuntu版本代号，第三个universe是软件仓库的区域，也就是要查看的代码仓库的不同分区。

    Ubuntu图形界面中software&update中可以直接切换软件仓库，更加方便

    终端包管理工具
    apt-get/apt管理包命令
    aptitude：这个命令再卸载软件时可以卸载不用的依赖
    sudo apt update更新软件包缓存；
    sudo apt-cache search搜索软件包；
    sudo apt install name安装软件包；
    sudo apt upgrade升级所有已安装软件包；
    sudo apt autoremove

    dpkg命令时apt-get和aptitude两个命令的后端（backend）命令

    依赖链：dpkg–>apt-get,aptitude–>Ubuntu Software

    使用dpkg可以安装本地的软件包：sudo dpkg -i *.deb，
    卸载：sudo dpkg -r 包名

    yum等傻瓜式安装 终端的软件包管理器 企业级不会用
    yum 是 CentOS 中的默认包管理器工具，也用于 Red Hat 一族
    优点: 安装简单，快捷
    缺点: 完全丧失了自定义性
    注意：如果不更改软件来源的情况下，是需要联网才能使用yum的。
    ls /etc/yum.repos.d/ 查看源下载
    常用的yum指令：

    • yum list [installed] 列出当前已经装的和可以装的软件（全部）
    • yum search 名 搜索指定的关键词的包
    • yum [-y] install 包名 安装指定的包（-y表示允许不再确认）
    • yum [-y] update [包名] 更新指定的包，不指定包则更新全部软件，删除旧包
    • yum [-y] upgrade [包名] 更新指定的包，不指定包则更新全部软件，不删除旧包
    • yum [-y] remove 包名 删除卸载指定的包
    • sudo yum autoremove xxx
    • yum clean packages 清理缓存
    • yum info xx 软件包信息

    yum remove firefox指令卸载火狐浏览器
    yum install firefox指令安装火狐浏览器

    本地的 *.rpm 软件包安装的命令

    1. sudo yum localinstall *.rpm 用于安装
      sudo yum remove *.rpm 用于卸载
    2. rpm -ivh 软件包完整名称
      选项：
    • -i：install，安装
    • ``-v`：显示进度条
    • -h：表示以“#”形式显示进度条

    rpm -e 软件的名称 卸载某个软件
    火狐卸载的时候是没有依赖关系的，所以可以直接卸载。
    但是在卸载Apache的时候提示无法卸载：
    当存在依赖关系的时候又不想去解决这个问题的时候可以：
    rpm -e 软件包名 --nodeps
    rpm -qa|grep 关键词 查询某个软件的安装情况
    Red-Hat Package Manager

    选项：
    -q：查询，query
    -a：全部，all
