    Python的多进程编程

    标准库有两个： Multiprocessing 和 Concurrent.futures
    先有的 multiprocessing，后有的 concurrent.futures，后者的出现就是为了降低编写代码的难度，后者的学习成本较低。

    在速度上，并无谁快谁慢之说。获得多少加速（如果有）取决于硬件，操作系统的详细信息，尤其取决于特定任务需要多少进程间通信。在后台，所有进程都依赖于相同的 OS 原语，使用这些原语的高级 API 并不是j影响速度的主要因素。接下来分享下二者的详细用法。

    multiprocessing多进程模块实现多进程，利用电脑多个CPU核心的运算能力

    • multiprocessing.Process(target=func, args=(1,))返回进程对象

      • target指向函数名，
      • args需要向函数传递的参数，元组，没有参数可不写

    • start()启动创建的新进程，无参数无返回值

    • kill()杀死进程，无参数无返回值

    • join()挂起阻塞主进程，等待对应的子进程完成，无参数无返回值

    • is_alive()无参数，返回布尔值，这个线程是否还存活

    • os.getpid()获得当前函数执行的进程的ID
      获取进程函数状态的函数：ready()、successful()、get()。

    在一个脚本中，创建一个子进程，目的是为了让它执行脚本中的某个函数，将脚本中的某一个函数让它单独启一个进程去执行，因为进程间互不干扰同时执行，主进程的程序和子进程里执行的函数互不干扰
    多进程中因为每个进程都能被系统分配资源，相当于每个进程有了一个python解释器，
    所以多进程可以实现多个进程的同时运行，缺点是进程系统资源开销大
    可以用多进程实现 cpu 密集程序，避免 GIL 的影响

    当多进程运行时，可能会出现的问题即解决方案

    • 多个进程同时修改文件可能会出现错误——进程间如何避免资源抢占：创建进程锁
    • 进程数量太多可能会造成资源不足，甚至死机等情况——如何避免创建进程数量过多：创建进程池
    • 通过进程模块执行的函数无法获取返回值，即使有return——进程间如何通信：通过队列
      不需要安排取哪些数据进程的数据可放入队列，哪些进程需要，从队列中通过取出，即可使用。

    运行后，打开任务管理器的详细信息，可以发现 有3个Python进程，其中主进程CPU占用率为0，两个子进程CPU各占满了一个核心的运算能力。

    都在唯一一个主进程下，work_b函数必须等待同一个进程下的work_a函数执行完之后才能执行

    work_a函数在子进程，work_b在主进程中运行，同时运行互不干扰

    work_a函数在子进程，work_b在子进程中运行，同时运行互不干扰

    利用multiprocess模块的Pool类创建进程池

    很多时候系统都需要创建多个进程以提高CPU的利用率，当数量较少时，可以手动生成一个个Process实例。
    当进程数量很多时，或许可以利用循环，但是这需要程序员手动管理系统中并发进程的数量，有时会很麻烦。
    这时进程池Pool就可以发挥其功效了。

    进程池本身在还没有执行完成之前，主进程已经结束程序退出，python解释器调用了system.except函数，而进程池其实也是主进程的一部分，
    进程池也就关闭了，进程池中的进程也就关闭了

    上面代码加上一句time.sleep(20)

    进程池本身在还没有执行完成之前，主进程还没结束，20秒给进程池足够的时间完成工作
    进程池的每个id不一样，有相同的进程号，先执行5个，过了5秒又执行5个，因为进程池只有5个进程，
    for循环有20个任务要处理，work函数要被执行20此，所以每一次这个进程池中最大的执行工作进程只有5，

    因为小编我的CPU是4核的，一次最多可以同时运行4个进程，所以我开启了一个容量为4的进程池。4个进程需要计算5次，你可以想象4个进程并行4次计算任务后，还剩一次计算任务(任务4)没有完成，系统会等待4个进程完成后重新安排一个进程来计算。

    队列是一种数据存储结构，它的数据存储特点类似于排队，
    先进入队列的会先出来，后进入队列的后出来，
    multiprocessing.Queue(mac_count)返回队列对象，队列最大可以传入多少信息，如果不传入参数默认是无限长度，几乎复制 queue.Queue
    使用队列queue来实现不同进程间的通信或数据共享，这一点和多线程编程类似。
    实现不同线程间数据共享的方法就是使用消息队列queue。不像列表，queue是线程安全的，

    分布式进程

    使用多进程的时候，可以将进程部署在多个主机节点上，Python的multiprocessing模块不但支持多进程，其中managers子模块还支持把多进程部署到多个节点上。当然，要部署分布式进程，首先需要一个服务进程作为调度者，进程之间通过网络进行通信来实现对进程的控制和调度，由于managers模块已经对这些做出了很好的封装，因此在无需了解网络通信细节的前提下，就可以编写分布式多进程应用。具体的请参照本章分布式多进程爬取“手机搜狐网”的实例代码。
