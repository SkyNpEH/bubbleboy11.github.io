    141. 环形链表

    给定一个链表，判断链表中是否有环。

    如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。
    为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

    如果链表中存在环，则返回 true 。 否则，返回 false 。

    分析
    不用 dummy

    链表不可能有多个环，因为一个节点不能有两个后继指针；
    可行方案：
    1、遍历是否能够出现 NIL 空指针；
    2、使用 Set 存储已访问过的节点，如果当前待遍历的节点已存储在Set集合中，则证明有环；

    给 2 个指针，一个指针是另外一个指针的下一个指针。

    「Floyd 判圈算法」（又称龟兔赛跑算法）

    假想「乌龟」和「兔子」在链表上移动，「兔子」跑得快，「乌龟」跑得慢。当「乌龟」和「兔子」从链表上的同一个节点开始移动时，如果该链表中没有环，那么「兔子」将一直处于「乌龟」的前方；如果该链表中有环，那么「兔子」会先于「乌龟」进入环，并且一直在环内移动。等到「乌龟」进入环时，由于「兔子」的速度快，它一定会在某个时刻与乌龟相遇，即套了「乌龟」若干圈。

    我们可以根据上述思路来解决本题。具体地，我们定义两个指针，一快一满。
    慢指针每次只移动一步，而快指针每次移动两步。只要是成环两者以不同步移动总会相遇，要是慢一快三未必可以解。可以将环理解为一个整数加法群，只有差为一才一定有解。如果相差的步数太多能会导致快指针直接越过慢指针，有些情况下永远都不会相遇。如果快减慢等于1，必相遇一定有解的。如果这个环只有两个节点,那好像就只能慢1,快2了

    假设快慢指针均位于 n 元环内，不妨设快指针比慢指针远 k 步，设快指针步长为 x，设 t 步相遇，那么列出方程组，x + k = tx mod n，即 (t - 1)x = k mod n，当且仅当 t - 1 和 n 互素时，可以计算 t - 1 在模 n 意义下逆元，进而解出 x，其他情况无解。快指针取 2 的好处在于一定有解，且最快相遇，因为 x = k mod n

    但因为是离散的,如果fast的步长和slow的步长改变,可能会无法相遇.可以参考poj 1061 青蛙的约会.
    步长每多一个，就要多加一个非空判断，没有必要把步长加大的。
    而且步长还得互质，不然永远不相遇

    初始时，慢指针在位置 head，而快指针在位置 head.next。这样一来，如果在移动的过程中，快指针反过来追上慢指针，就说明该链表为环形链表。否则快指针将到达链表尾部，该链表不为环形链表。

    为什么我们要规定初始时慢指针在位置 head，快指针在位置 head.next，而不是两个指针都在位置 head（即与「乌龟」和「兔子」中的叙述相同）？

    观察下面的代码，我们使用的是 while 循环，循环条件先于循环体。
    由于循环条件一定是判断快慢指针是否重合，如果我们将两个指针初始都置于 head，那么 while 循环就不会执行。因此，我们可以假想一个在 head 之前的虚拟节点，慢指针从虚拟节点移动一步到达 head，快指针从虚拟节点移动两步到达 head.next，这样我们就可以使用 while 循环了。

    时间复杂度：O(N)，其中 N 是链表中的节点数。
    当链表中不存在环时，快指针将先于慢指针到达链表尾部，链表中每个节点至多被访问两次。
    当链表中存在环时，每一轮移动后，快慢指针的距离将减小一。而初始距离为环的长度，因此至多移动 N 轮。

    空间复杂度：O(1)。我们只使用了两个指针的额外空间。

    例子
    3 -> 2 -> 0 -> 4 -> 2
    head = [3,2,0,-4]

    if head == nil || head.Next == nil {
    head = 3 != nil || head.Next = 2 != nil {

    slow, fast := head, head.Next
    slow = 3
    fast = 2
    fast = 3 != slow = 2

    slow = slow.Next = 3.Next = 2
    fast = fast.Next.Next = 2.Next.Next = -4 != nil
    fast.Next = -4.Next = 2 != nil

    slow = slow.Next = 2.Next = 0
    fast = fast.Next.Next = 2.Next.Next = 0 != nil

    fast = 2 = slow

    return true

    法二
    使用 do-while 循环。此时，我们就可以把快慢指针的初始值都置为 head。

    要确保fast以及fast.next存在，这样fast指针才能走两步。
    如果直接使用fast.next.next，如果fast以及fast.next不存在，会直接报错。
    又因为此处fast比slow快，所以可以不用加slow

    判断fast是因为fast要走两步，所以fast本身也就是链表本身非空和链表的下一步非空就可以往下走，不然fast就走不了，而且这样如果链表刚好是空或者是只有一个元素就不会进入循环，直接返回false了
