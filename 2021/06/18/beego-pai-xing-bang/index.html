beego排行榜

发表于 2021-06-17 更新于 2021-06-21 分类于 beego

排行榜功能分为两部分

图一，频道页中动漫排行模块
图二，点击页面顶部热搜榜，打开排行榜页面

controllers\top.go

频道排行榜接口

接口地址：

/channel/top

接受参数和返回信息

接收参数为：channelId（频道ID，1=动漫）
返回信息：
id = 视频ID
title = 标题
sub_title = 副标题
img = 竖版图片地址
img1 = 横版图片地址
add_time = 添加时间
episodes_count = 剧集集数
is_end = 是否完结
comment = 评论数

业务逻辑：获取video表中channel_id=接收参数和status=1的数据，按照评论数倒序，获取10条。

类型排行榜接口

接口地址：

/type/top

接受参数和返回信息

接收参数为：typeId（类型ID）
返回信息：
id = 视频ID
title = 标题
sub_title = 副标题
img = 竖版图片地址
img1 = 横版图片地址
add_time = 添加时间
episodes_count = 剧集集数
is_end = 是否完结
comment = 评论数

业务逻辑：获取video表中type_id=接收参数和status=1的数据，按照评论数倒序，获取10条。

注意：本节排行榜功能是通过mysql实现的，对应条件倒序就可以了，但是假如数据量比较大的时候，接口访问速度肯定很慢。怎么优化和改造呢？小伙伴可以思考一下，先做铺垫，后面章节对应讲解。
