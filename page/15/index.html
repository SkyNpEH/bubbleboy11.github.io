切片 冒号适用于列表、字符串
切片[][1:3]还是list
[起始位置，长度]；
截取从起始位置，长度个字符；
长度负数表示删除从字符串末尾往前数的字符个数；
长度省略表示到末尾； 长度要是超过可截取的位数，以可截取的位数为标准；
单一的字符串索引操作得到的是字符串，
但是一个列表的切片操作得到的是一个列表

a=range（100）
range(start, end, step)，start：起始值；end：终值；step：步长，默认为1。
a：0、1、2、3、4、5、……99。
所以，
正索引范围为：a[0] -a[end-1]，
即 a[0] = 0，a[1] = 1，……，依次类推；
负索引范围为：a[-1] -a[-end]，
即 a[-1] = 99，a[-2] = 98，……，依次类推。

加解密算法，是对源数据 进行运算产生加密数据，以及反向过程，对加密数据反算出 源数据。

加解密算法 和 hash算法 不同点有：

  • 加解密算法 是可逆的，hash算法是不可逆的。
  • hash算法可以对很大的数据产生比较小的哈希值，而加密算法源数据很大，加密后的数据也会很大

加密方式

  • 对称加密：加密和解密的密钥是相同的，速度快，一般对文件，AES, RC4, DES, 3DES, IDEA
  • 非对称加密：公钥加密——私钥解密，私钥加密——公钥解密，强度高速度慢，互联网 RSA (Rivest–Shamir–Adleman) DSA ECC

协程

发表于 2020-08-20 更新于 2021-04-13 分类于 python

协程：协同多任务

  • 协程是轻量级线程，单线程，分解一个线程成为多个 “微线程”。
  • 可以执行并发任务，原因是协程把切换的权利交给程序员调度而不是操作系统，与程序员决定在哪些环节进行切换。协程可以处理上万的并发，多线程即不可以，因为切换成本太大，会耗尽计算机资源
  • 不需要锁机制，因为是单线程
  • 对多核CPU的利用——多进程+协程
    协程是为非抢占式多任务产生子程序的计算机程序组件，协程允许不同入口点在不同位置暂停或开始执行程序。从技术的角度来说，“协程就是你可以暂停执行的函数”

基于生成器的协程 Generator Based Coroutine

python3.5之前没有原生协程，是基于生成器（yield）来实现的协程。

  • pep 342(Coroutines via Enhanced Generator)增强生成器功能
  • 生成器可以通过yield暂停执行和产出数据
  • 同时支持send()向生成器发送数据和throw()向生成器抛异常
  • 协程需要使用 send(None) 或者 next(coroutine) 来预激 (prime)才能启动
  • 在 yield处协程会暂定执行
  • 单独yield value 会产出值给调用方
  • 可以单独coroutine.send(value) 来给协程发送值，发送的值会赋值给 yield表达式左边的变量
  • 协程执行完后（没有遇到下一个yield语句）会抛出StopIteratio异常

调用next()，函数将不断的执行语句直到遇到下一个yield的位置，yield在函数语句执行暂停，直到下一次遇到又会暂停

可变参数

函数传递中 *args, **kwargs 含义是什么

  • 用来处理可变参数允许定义的函数接受任意数目的参数。在函数被调用前并不知道也不限制将来函数可以接收的参数数量，将不定数量的参数传递给一个函数。

写成 *args 和 **kwargs 只是一个通俗的命名约定,主要用于函数定义。
其实并不是必须写成*args 和 **kwargs。
只有变量前面的 * 才是必须的.
你也可以写成 *var 和 **vars.

参数定义的顺序

必选参数、默认参数、可变参数、命名关键字参数和关键字参数。
func(fargs, *args, **kwargs) 规定顺序
func1(a, **kwargs, *args) SyntaxError: invalid syntax

位置参数 *args (positional argument)

在参数前面加了一个*
被打包成 tuple 元组
和定义一个list或tuple参数相比，
*args表示把args这个list的所有元素作为可变参数传给可变参数

既可以直接传入：func(1, 2, 3)，
再通过*args传入：func(*(1, 2, 3))；

关键字可变参数 **kwargs (keyword argument)

被打包成 dict 字典
在参数前面加了两个*，
**extra表示把extra这个dict的所有key-value用关键字参数传入到函数的**kw参数，
kw获得的dict是extra的一份拷贝，对kw的改动不会影响到函数外的extra。

可变参数和不可变参数混用实例

命名的关键字参数是为了限制调用者可以传入的参数名，同时可以提供默认值。
定义命名的关键字参数在没有可变参数的情况下不要忘了写分隔符*，否则定义的将是位置参数。

哈希hashlib MD5

发表于 2020-08-20 更新于 2021-05-10 分类于 python

摘要算法又称哈希算法、散列算法。它通过一个函数，把任意长度的数据（字节串）转换为一个长度固定的数据串（通常用16进制的字符串表示）。

在python3的标准库中，已经移除了md5模块，而关于hash加密算法都放在hashlib这个标准库中，
如SHA1、SHA224、SHA256、SHA384、SHA512和MD5算法等。

MD5消息摘要算法（Message Digest Algorithm 5），一种被广泛使用的密码散列函数，
可以产生出一个128位（16字节）的 哈希值（hash value）或者 摘要(digests)），用于确保信息传输完整一致。
通常用一个32位的16进制字符串表示。

哈希计算的特点是：

  • 相同的 源数据， 采用 相同的哈希算法， 计算出来的哈希值 一定相同
  • 不管 源数据 有多大，相同的哈希算法，计算出来的哈希值长度 都是一样长的。

算法 计算结果长度
MD5 16字节
SHA1 20字节
SHA224 28字节
SHA256 32字节
SHA384 48字节
SHA512 64字节

  • 算法不可逆。不能通过 哈希值 反过来计算出 源数据。 所以哈希和我们常说的加密解密不同。
  • 不同的源数据 使用同样的哈希算法，可能会产生相同的 哈希值，这被称之为碰撞率（collision rate）

回溯算法

发表于 2020-08-20 更新于 2021-04-02 分类于 数据结构与算法

回溯算法实际上一个类似枚举的搜索尝试过程，
主要是在搜索尝试过程中寻找问题的解，
当发现已不满足求解条件时，就 “回溯” 返回，尝试别的路径。
回溯法是一种选优搜索法，按选优条件向前搜索，以达到目标。
但当探索到某一步时，发现原先选择并不优或达不到目标，就退回一步重新选择，
这种走不通就退回再走的技术为回溯法，

回溯算法的基本思想是：从一条路往前走，能进则进，不能进则退回来，换一条路再试。

回溯算法建立在DFS基础之上的，但不同的是在搜索过程中，达到结束条件后，恢复状态，
回溯上一层，再次搜索。
而满足回溯条件的某个状态的点称为 “回溯点”。

DFS是一个劲的往某一个方向搜索，

因此回溯算法与DFS的区别就是有无状态重置

** 当问题需要”回头”，以此来查找出所有的解的时候 **，使用回溯算法。
即满足结束条件或者发现不是正确路径的时候(走不通)，要撤销选择，回退到上一个状态，
继续尝试，直到找出所有解为止
许多复杂的，规模较大的问题都可以使用回溯法，有“通用解题方法”的美称。

①画出递归树，找到状态变量(回溯函数的参数)，这一步非常重要※
②根据题意，确立结束条件
③找准选择列表(与函数参数相关),与第一步紧密关联※
④判断是否需要剪枝
⑤作出选择，递归调用，进入下一层
⑥撤销选择
