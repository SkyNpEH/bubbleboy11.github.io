Python使用多线程
应用程序必须 通过操作系统提供的 系统调用，请求操作系统分配一个新的线程。线程执行代码都是在cpu中执行的

当多个线程运行时，可能会出现的问题及解决方案

  • 通过线程执行的函数无法获取返回值——线程间如何通信：通过队列
  • 多个线程同时修改文件可能造成数据错错乱——线程间如何避免资源抢占：创建线程锁
  • 线程数量太多可能会造成资源不足，甚至死机等情况——如何避免创建线程数量过多：创建线程池

实现主线程和子线程的同步

如果一个主线程A的代码调用了 线程B的Thread对象的 join() 方法，直至启动的子线程B终止之前主线程A就会停止继续执行阻塞挂起调用该方法的线程，直到被调用join()方法的线程B执行完毕退出后结束后，线程A才继续执行后续的代码。

解决了一个线程需要等待其它的线程结束，比如需要根据其他线程运行结束后的结果进行处理。
主线程把任务分配给几个子线程，等待子线程完成工作后，需要对他们任务处理结果进行再处理。
就好像一个领导把任务分给几个员工，等几个员工完成工作后，他需要收集他们的提高报告，进行后续处理。

而子线程执行的代码如果不能正常退出时那主线程也一直会等待，
在join时加上一个超时时间(timeout=None)timeout（秒），那主线程只等待设置的超时时间后主线程就退出，否则会一直阻塞，

主线程必须等待参数对应的子线程完成才能进行后续操作

这说明当一个线程是none daemon时，并调用了join()函数，那主线程会等待子线程执行完成。这和不调用join()函数的none daemon线程是有区别的，调用join()函数时是主线程等待子线程完成，不调用join()函数时是主线程被阻塞后，一直等待子线程执行完成，从打印的日志可看出两者的区别。

在none daemon的线程中，如果给join()函数一个超时时间，当超过这个时间后，即使这个线程没有执行完成，程序就直接输出了2021-04-04 16:25:35,343 DEBUG [MainThread] main thread exiting ，并且被阻塞在这里等待子线程的完成，因为线程是none daemon的。这种场景在现实的编程中应该不多。

  • 信号量(Semphare)：控制同一时刻多个线程访问同一资源的线程数
  • 事件(信号)：通过通知的方式保持多个进程同步

Lock 锁

原始锁处于”锁定“或者”非锁定“两种状态之一。它被创建时为非锁定状态。Lock 对象有两个基本方法，

  • threading.Lock().acquire():
    • 当状态为非锁定时: acquire()将状态改为锁定 并立即返回。
    • 当状态是锁定时，acquire() 将阻塞至其他线程调用release()将其改为非锁定状态，然后 acquire() 调用重置其为锁定状态并返回。是申请锁。
  • threading.Lock().release() 只在锁定状态下调用； 它将状态改为非锁定并立即返回。如果尝试释放一个非锁定的锁，则会引发RuntimeError异常。

3.非原子的操作不是线程安全的，通过加锁来保证线程安全。

Barrier 栅栏
栅栏类提供一个简单的同步原语，用于应对固定数量的线程需要彼此相互等待的情况。线程调用wait()方法后将阻塞，直到所有线程都调用了wait()方法。此时所有线程将被同时释放。

形象理解：像全班组织出去春游，大家都要到指定地点（栅栏）进行等待，等人到齐，一起出发。

wait(timeout=None)：冲出栅栏。当栅栏中所有线程都已经调用了这个函数，它们将同时被释放。
reset()：重置栅栏为默认的初始态。
abort()：将栅栏置为破损态。会导致所有的active或者future等待失败，并收到BrokenBarrierError。
parties：冲出栅栏所需要的线程数量。
n_waiting：当前时刻正在栅栏中阻塞的线程数量。
broken：表示栅栏是否为破损态。

threading.local类

使用线程时最不愿意遇到的情况就是多个线程竞争资源，在这种情况下为了保证资源状态的正确性，我们可能需要对资源进行加锁保护的处理，这一方面会导致程序失去并发性，另外如果多个线程竞争多个资源时，还有可能因为加锁方式的不当导致死锁。要解决多个线程竞争资源的问题，其中一个方案就是让每个线程都持有资源的副本（拷贝），这样每个线程可以操作自己所持有的资源，从而规避对资源的竞争。

要实现将资源和持有资源的线程进行绑定的操作，最简单的做法就是使用threading模块的local类，在网络爬虫开发中，就可以使用local类为每个线程绑定一个MySQL数据库连接或Redis客户端对象，这样通过线程可以直接获得这些资源，既解决了资源竞争的问题，又避免了在函数和方法调用时传递这些资源。具体的请参考本章多线程爬取“手机搜狐网”（Redis版）的实例代码。

Python的多进程编程

  • multiprocessing多进程模块实现多进程，利用电脑多个CPU核心的运算能力

  • multiprocessing.Process(target=func, args=(1,))返回进程对象

    • target指向函数名，
    • args需要向函数传递的参数，元组，没有参数可不写

  • start()启动创建的新进程，无参数无返回值

  • kill()杀死进程，无参数无返回值

  • join()挂起阻塞主进程，等待对应的子进程完成，无参数无返回值

  • is_alive()无参数，返回布尔值，这个线程是否还存活

  • os.getpid()获得当前函数执行的进程的ID

在一个脚本中，创建一个子进程，目的是为了让它执行脚本中的某个函数，将脚本中的某一个函数让它单独启一个进程去执行，因为进程间互不干扰同时执行，主进程的程序和子进程里执行的函数互不干扰

多进程中因为每个进程都能被系统分配资源，相当于每个进程有了一个python解释器，
所以多进程可以实现多个进程的同时运行，缺点是进程系统资源开销大
Python有GIL，可以用多进程实现 cpu 密集程序，避免 GIL 的影响

当多进程运行时，可能会出现的问题即解决方案

  • 多个进程同时修改文件可能会出现错误——进程间如何避免资源抢占：创建进程锁
  • 进程数量太多可能会造成资源不足，甚至死机等情况——如何避免创建进程数量过多：创建进程池

xrange(stop)
xrange(start, stop[, step])

start: 计数从 start 开始。默认是从 0 开始。
例如 xrange(5) 等价于 xrange(0， 5)

stop: 计数到 stop 结束，但不包括 stop。
例如：xrange(0，5) 是 [0, 1, 2, 3, 4] 没有 5

例如：xrange(0，5) 等价于 xrange(0, 5, 1)

Event 事件

多线程间通信的一种简单机制：一个线程发出事件event信号，而其他线程等待该信号。一个事件对象管理一个内部标志。

  • is_set()：内部标志是否为True
  • set()：将内部标志设为True
  • clear()：将内部标志设为False
  • wait(timeout=None)：阻塞直到线程内部标志为True，即有线程调用set()方法。
    设置超时时间，表示只阻塞线程一定的时间，如果收到了event.set()信号，线程退出，否则线程被阻塞，只是不是一直阻塞，而是按照设定的时间阻塞线程，

Condition 条件对象

confition方法适合生产-消费的工作模型，消费者会等待生产者生产数据生消费者再对数据进行处理。

  • acquire()：请求底层锁。
  • release()：释放底层锁。
  • wait(timeout=None)：等待直到被通知或发生超时。如果线程在调用此方法时没有获得锁，将会引发RuntimeError异常。
  • wait_for(predicate, timeout=None)：等待，直到条件计算为真。
  • notify()：默认唤醒一个等待这个条件的线程。
  • notify_all()：唤醒所有正在等待这个条件的线程。

fast = fast.next.next
if slow == fast:
return True
