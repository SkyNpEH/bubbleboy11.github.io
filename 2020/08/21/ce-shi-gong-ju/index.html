fiddler

http协议调试代理工具，它能够记录并检查所有你的电脑和互联网之间http通讯，设置断点，查看所有的“进出”fiddler的数据。
可以手机抓包，模拟弱网，拦截请求，定制响应
fiddler的工作原理：在电脑端设置一个代理，电脑发起的所有http请求都会经过这个代理，fiddler代理把收到的请求后再将这个请求发给服务器，服务器收fiddler的请求处理后在返回给fiddler，fiddler收到服务端返回的数据同样再返回给我们的电脑

打开后，系统代理变成默认端口8888，
选择Connections，fiddler监听的端口默认是8888，如果该端口与你其他应用的端口冲突，你也可以自定义1024~65536范围内的任意端口，然后勾选作为系统代理。

Postman

自动反复测试一个需要携带header的post请求，并且对针对线上、测试环境同一个数据进行测试。
并对实际结果做比较，以作为测试结果
关键字：自动 反复 线上 线下

Postman是一种网页调试与发送网页http请求的chrome插件。我们可以用来很方便的模拟get或者post或者其他方式的请求来调试接口

使用Postman如果通过自己拼接url，自己造数据，是非常麻烦的。因此选择Postman+Postman interceptor，使用Postman interceptor可以拦截到网页请求，同时传到postman的history列表中，然后我们只需要在postman中修改一下url就可以直接send了。

主要功能包括

  • 模拟各种 HTTP requests

  • Collection 功能（测试集合）栏保存有常用的接口，可以按接口类型或者业务类型放在不同的文件夹下。
    把接口放到collections里同一个文件夹下可以进行接口的批量执行，需要把所有要测试的接口放在一个文件夹下，点击右边的三角形，然后点击run，进行运行
    iteration：迭代次数

  • Params url带上参数

  • Authentication 认证

    • Basic Auth 用户名密码认证
    • API Key: Token认证
    • OAuth 2.0:

  • 需要header或者cookie时候，则要填写header才可以成功请求，cookie的值也是填写到header中，添加cookie和header之后添加入参到Body中
    添加key value

  • body里有内容，get会把数据都放到url中，body不可以点击。可以加表单，可以上传txt和file。
    在后端解析的代码解析的request.data通常放在 raw JSON 里面，
    更新post,body里面修改发送

  • 人性化的 Response 整理

  • 内置测试脚本语言，点击右边的code，可以选择自己熟悉的代码，选择之后，根据选择生成接口测试的代码。 生成对应语言的http请求的代码。

  • 设定变量与环境 enviroment：右上角的设置按钮，选择manage environment，输入环境名称和url
    如设置线上和测试环境的地址名称，这样可以自由地切换不同的测试环境来执行接口测试。

  • 通过tests栏设置相关内容可以判断测试结果是pass还是fail。
    在返回数据查uid，如果查到了就说明接口是成功的
    tests["Body matches string"]=responseBody.has（"uid"）;
