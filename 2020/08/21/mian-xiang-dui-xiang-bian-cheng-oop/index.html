面向对象编程OOP

发表于 2020-08-20 更新于 2021-05-08 分类于 python

面向对象编程 Object Oriented Programming（OOP）
设计思想是 描述现实世界
把对象作为基本单元，把对象抽象成类(Class)，包含成员(属性)和方法(用来操作成员)。
根据Class创建Instance。

类Class是一种抽象模板，类是现实世界或思维世界中的实体在计算机中的反映，
它将数据以及这些数据上的操作封装在一起，概念

对象：通过类定义的数据结构实例。对象包括两个数据成员（类变量和实例变量）和方法。值 身份id 类型

面向对象三大特点：

  • 封装：对外部隐藏有关对象工作原理的细节，提供一些可以被外界访问的属性的方法
  • 继承：使用现有类的所有功能，并在无需重新编写原来的类的情况下对这些功能进行扩展
  • 多态：子类重写父类方法

Python 中使用类来实现。过程式编程(函数)，OOP(类)
(object)，表示该类是从哪个类继承下来的，
默认继承object类，可不写
创建类

定义实例方法（函数）：
实例方法关联的是对象，实例方法通常用来操作实例变量，
def __init__(self, name, *args,**kwargs): 可以通过外部访问，是因为init前后分别有两个下划线‘__’, 这是python的内置命名风格，自己命名尽量不要这样做
第一个形参（实例变量）不是必须叫self，self 不是 python 关键字，（其他语言用this，其他和普通函数一样）
但是第一个参数代表的是表示创建类的实例本身（对象），代表当前对象的地址，
把各种属性绑定到self，
在传参数时不需要给self传值，Python解释器自己会把实例变量传进去，

实例方法通过self.xxx访问实例变量和类变量，具体区分参考上文，不推荐
实例方法可以通过self.class.xxx访问类变量
实例方法不可以直接访问类变量

self.class 则指向类。

对象名=类名([实参]) 对象是类的实例化，形参写在类的构造函数中，
实例（Instance）则是根据类模板创建出来一个个具体的“对象”
各个实例拥有的数据都互相独立，互不影响；

每个实例化的对象当使用完成之后，都会被内存管家回收，我们普通的进程也会伴随着创建与关闭的过程
Python严格要求方法需要有实例才能被调用，这种限制其实就是Python所谓的绑定概念

变量名前加单下划线_表示的是 protected 类型的变量，即保护类型只能允许其本身与子类进行访问，如：self._sex = 0 。
变量名前加双下划线__表示的是 私有类型(private)的变量,
成员私有 类的私有属性
在变量或方法名前加__，则变为私有private私有属性，在类外部无法直接进行访问、设置
只能是允许这个类本身内部的方法进行访问了。如：self.__foo = 0 self.__private_attrs。

类的私有方法
__private_method：两个下划线开头，声明该方法为私有方法，只能在类的内部调用 ，不能在类的外部调用。self.__private_methods。

在创建私有的实例变量时，实际python会把变量名改为_类名__变量名，
所以在外部访问私有变量__变量名时，实际是创建了新变量_类名__变量名。
严格来讲python并没有真正的私有变量，因为可以通过对象名._类名__变量名，可以修改和访问私有变量

在方法或者变量前加__

__dict__ 在类中代表类的全部属性的字典，在实例中代表对象的全部属性的字典
print(student.dict)
print(Student.dict)

2.classmethod / staticmethod区别 (它们两个经常被用作类方法的装饰器)

  • 都可以通过 Class.method() 的方式调用

  • 类方法：定义加@classmethod，类方法关联的是类，通常用来操作类变量，
    第一个形参是 cls 代表的是调用的类本身，类方法不会被自动调用，（也可以是别的，建议cls）
    类方法可以通过类名.方法名()和实例对象名.方法名()两种方法进行调用 （但是不建议用对象调用类方法，毕竟逻辑不符），
    在类方法中访问类变量，用cls.变量名访问；
    在类方法中不可以访问实例变量。

  • staticmethod 静态方法，使用起来和普通函数一样，只不过放在类里去组织,
    是代码组织的需要，完全可以放到类之外
    并没有引用当前类的任何东西，可以把staticmethod理解为本来该放到外面的函数只是给塞到类里。其实跟类并没有什么关系，相当于给了它一个作用范围，让它可以在类里面去使用
    类的静态方法可以通过类名.方法名()和实例名.方法名()调用
    与类方法不同的是可以不用传第一个参数
    访问类变量通过类名.变量名
    类方法和静态方法不能访问实例方法
